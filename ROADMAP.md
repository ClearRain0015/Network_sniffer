# 后续改进路线图

## 一、协议解析（当前 12 种 → 目标 20+）

- [ ] **TLS/SSL 解析** — 识别 ClientHello / ServerHello 握手信息（不涉及解密）
- [ ] **QUIC / HTTP3 解析** — UDP 443 端口识别
- [ ] **SSH 解析** — 端口 22，提取版本协商
- [ ] **FTP 解析** — 端口 21，控制通道命令解析
- [ ] **SMTP 解析** — 端口 25，邮件传输
- [ ] **NTP 解析** — 端口 123，时间同步
- [ ] **SNMP 解析** — 端口 161，网络管理
- [ ] **ARP 欺骗检测** — 同一 IP 多个 MAC 的异常告警

## 二、抓包能力

- [ ] **混杂模式** — 抓取非本机的数据包（需要 Npcap 开启混杂）
- [ ] **PCAP 文件导入** — 打开已有 `.pcap` / `.pcapng` 文件回放分析
- [ ] **抓包轮转保存** — 每 N 个包/每 N MB 自动切割新文件
- [ ] **BPF 内核级过滤** — 将过滤表达式下推到抓包驱动层，减少丢包

## 三、显示过滤

- [ ] **字段级过滤** — 支持 `tcp.srcport == 80`、`ip.ttl < 64` 等
- [ ] **比较运算符** — `==` `!=` `>` `<` `>=` `<=` `contains`
- [ ] **更多组合** — `()` 分组、复杂嵌套表达式

## 四、会话分析

- [ ] **TCP 流跟踪（Follow TCP Stream）** — 将一次 TCP 会话的收发数据拼成完整对话
- [ ] **UDP 流跟踪（Follow UDP Stream）** — 同上
- [ ] **HTTP 请求/响应配对** — 将 Request 和 Response 关联显示
- [ ] **DNS 请求/响应配对** — 同上
- [ ] **按会话着色** — 同一 5 元组的包用相同背景色

## 五、网络诊断

- [ ] **RTT 计算** — 根据 TCP 三次握手时间差估算往返时延
- [ ] **重传检测** — 标记重复 Seq 号的 TCP 包
- [ ] **零窗口检测** — 标记 Window=0 的 TCP 包
- [ ] **专家信息面板** — 自动标注可疑/异常包（重传、校验和错误、分片等）

## 六、统计图表

- [ ] **吞吐量时序图（IO Graph）** — 可叠加多条曲线，自定义粒度
- [ ] **会话流量排名图** — 按 IP/端口统计的柱状图
- [ ] **抓包实时仪表盘** — 类似任务管理器的实时刷新面板

## 七、数据操作

- [ ] **包注入/重放** — 选中某个包，修改字段后重新发送
- [ ] **PCAP 去重/合并** — 多个 pcap 文件合并、去重
- [ ] **包注释/书签** — 给包打标签、备注

## 八、GUI 体验

- [ ] **列排序** — 点击表头按某列排序
- [ ] **列自定义** — 显示/隐藏列、调整顺序
- [ ] **包搜索/跳转** — Ctrl+F 搜索包内容
- [ ] **时间显示格式切换** — 绝对时间 / 相对时间 / 增量时间
- [ ] **暗色模式** — 完整的暗色主题
- [ ] **多语言支持** — 中英文切换

## 九、性能

- [ ] **大文件优化** — 10 万+ 包时的虚拟列表渲染
- [ ] **多线程解析** — 解析链并行化，减少丢包
- [ ] **内存限制** — 可配置最大缓存包数，超出自动丢弃旧包
- [ ] **exe 体积优化** — 排除 matplotlib（~40MB），单独提供统计工具

## 主要不足（剩余差距）

| # | 不足 | 对比 |
|---|------|------|
| 1 | 协议种类少 | 12 种 vs Wireshark 1000+，缺 TLS/SSL、QUIC |
| 2 | 无内核级 BPF 过滤 | 高流量时可能丢包 |
| 3 | 无 TCP 重传/RTT 诊断 | 缺乏网络质量分析 |
| 4 | 不能包重放/注入 | 无法做网络实验 |
| 5 | 无会话时序分析 | 不能看流量随时间变化的热力图 |
| 6 | 不能合并 PCAP | 多文件分析不便 |

## 独特优势（超越 Wireshark）

| # | 优势 | 说明 |
|---|------|------|
| 1 | **SYN 洪水自动告警** | 原生实时检测，Wireshark 需插件 |
| 2 | **一键打包 exe** | 无需 Python 环境，下载即用 |
| 3 | **单元测试覆盖** | 工业级测试保证，Wireshark/tcpdump 均无 |
