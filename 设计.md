# Sniffer 网络数据包嗅探器 — 课程设计文档

## 前言

这是一个模块化的 Sniffer（网络嗅探器），而非"简化版 Wireshark"。老师给的要求对应的是网络分析器的基本架构，完全可以按照真正的软件工程去设计。需求里已经包含了所有模块，只需要重新组织即可。

## 四、每个模块做什么

### ① 网卡管理模块

```
列出所有网卡
    ↓
用户选择
    ↓
打开网卡
    ↓
开始监听
```

Python 可以直接用 `scapy.get_if_list()` 得到 Ethernet / WLAN / VMware / Loopback。

### ② 数据捕获模块

```
持续监听
    ↓
收到一个 Packet
    ↓
交给解析模块
```

核心循环：

```python
while True:
    packet = next_frame()
    Parser.parse(packet)
    GUI.update()
```

整个程序实质上就是不断重复这个过程。

### ③ 协议解析模块（重点）

**协议树：**

```
Ethernet
    ├── ARP
    ├── IPv4
    │   ├── ICMP
    │   ├── TCP
    │   │   ├── HTTP
    │   │   ├── HTTPS
    │   │   └── FTP
    │   └── UDP
    │       ├── DNS
    │       └── DHCP
    └── IPv6（可选）
```

**解析顺序：**

```
Ethernet → 判断Type → ARP? / IPv4?
    → 读取Protocol → TCP? / UDP? / ICMP?
```

每层负责解析自己的协议。

#### Ethernet 解析字段

- Destination MAC
- Source MAC
- EtherType

#### IPv4 解析字段

- Version
- Header Length
- TTL
- Total Length
- Identification
- Flags
- Fragment Offset
- Protocol
- Checksum
- Source IP
- Destination IP

> 老师一般都会检查这里。

#### TCP 解析字段

- Source Port
- Destination Port
- Sequence Number
- ACK Number
- Flags
- Window Size

**Flags 全部显示：** SYN、ACK、FIN、RST、PSH、URG

#### UDP 解析字段

- Source Port
- Destination Port
- Length
- Checksum

#### ICMP 解析字段

- Echo Request / Echo Reply
- Type
- Code
- Checksum

> Ping 时应该能看到。

#### ARP 解析字段

- Sender MAC
- Target MAC
- Sender IP
- Target IP
- Opcode

> 浏览网页时 ARP 会大量出现。

## 五、IP 分片重组（课程重点）

> **老师要求：数据报文重组**

这个模块一定要单独写。

### 设计思路

```
Fragment Table
key: (Source IP, Destination IP, Identification)
    ↓
收到一个 Fragment
    ↓
加入缓存
    ↓
全部收到
    ↓
排序
    ↓
拼接 Payload
    ↓
输出完整 IP 包
```

### 数据结构

```python
fragment_table = {
    (src_ip, dst_ip, id): [
        fragment1,
        fragment2,
        fragment3,
    ]
}
```

### 判断依据

- MF = 0 &nbsp;（最后一个分片）
- Offset 全部到齐 → 重组

> 这部分会成为老师问答重点。

## 十、建议实现顺序（按阶段开发）

### 第一阶段：抓包基础 ✓

- 枚举网卡
- 选择网卡
- 开始 / 停止抓包
- 在终端打印数据包基本信息

### 第二阶段：协议解析

- 解析 Ethernet
- 解析 IPv4
- 解析 ARP
- 解析 TCP / UDP / ICMP
- 显示十六进制数据和可读 Payload

### 第三阶段：GUI

- 使用 Tkinter（简单）或 PyQt5（更专业）
- 实现数据包列表、详细信息和十六进制数据显示
- 支持实时刷新

### 第四阶段：高级功能

- BPF 过滤器（按协议、IP、端口过滤）
- IP 分片重组
- 保存为 .pcap
- 流量统计（协议分布、Top IP、数据包数量等）

### 第五阶段：扩展（加分项）

- HTTP 请求头解析
- DNS 查询解析
- 流量趋势图
- 实时告警（例如检测大量 SYN 包）
